## Design Context

### Users

Developers using AI coding assistants (Claude Code, Copilot, Cursor, etc.) who need to manage MCP server configurations, skills, and agent settings. They value precision, control, and efficiency. Context: desktop app, workflow tool, technical configuration management.

### Brand Personality

**Precise, trustworthy, developer-focused**

Voice: Clear, concise, no fluff. Trust through transparency and control.
Emotional goals: Confidence and calm. Users should feel in control of their AI tooling.

### Aesthetic Direction

**Minimal and clean, friendly and utilitarian, NOT warm**

Reference: Linear — clean layouts, purposeful spacing, subtle interactions, functional elegance
Anti-reference: Apple-esque minimalism (too soft, too "designed", not utilitarian enough)

Keep the mint/green accent (OKLCH hue 155) — it's distinctive and functional.

### Design Principles

1. **Information density over whitespace** — Developers need to see what's relevant without scrolling
2. **Clear hierarchy through structure, not decoration** — Let layout communicate importance
3. **Utility-first, delight-second** — Every visual element must earn its place
4. **Transparency builds trust** — Show the underlying data, don't hide complexity
5. **Calm through consistency** — Predictable patterns reduce cognitive load
6. **Friendly but not playful** — Approachable without being cutesy

### Color Strategy

- Keep mint/green accent (--accent: oklch(82% 0.12 155))
- Use color purposefully: green for success/actions, reds for danger, minimal decorative color
- Avoid warm tones (oranges, warm yellows, browns)
- Maintain high contrast for readability

### Typography

- Inter (current) is acceptable — clean, utilitarian, developer-friendly
- Focus on hierarchy through size and weight, not decorative fonts
